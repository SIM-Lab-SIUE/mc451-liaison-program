# \ud83d\uddc2\ufe0f Resources - Your Media Library\n\nThis folder stores all **attachments**\u2014images, PDFs, screenshots, datasets, and any other files you reference in your notes.\n\n## What Goes Here\n\n- \u2705 Screenshots from lectures\n- \u2705 PDFs of journal articles\n- \u2705 Images for your project report\n- \u2705 Exported charts from R/ggplot2\n- \u2705 CSV datasets\n- \u2705 Slides and handouts\n- \u2705 Any media files you need to reference\n\n---\n\n## Why This Folder Matters\n\n### Problem Without It:\nYou embed an image in a note:\n```markdown\n![Chart](C:\\Users\\YourName\\Desktop\\random_folder\\chart.png)\n```\n\nThen you:\n- Move the file \u2192 Link breaks \u274c\n- Share your vault \u2192 Image missing for others \u274c\n- Sync to another device \u2192 Path doesn't exist \u274c\n\n### Solution With It:\n```markdown\n![Chart](chart.png)\n```\n\nObsidian knows to look in `04_Resources/` automatically. \u2705\n\n---\n\n## Setup (Required!)\n\n### Step 1: Configure Obsidian\n1. Go to **Settings** (gear icon)\n2. Click **Files & Links**\n3. Under \"Default location for new attachments\", select **`04_Resources`**\n4. Save\n\nNow every image/file you drag into Obsidian goes here automatically.\n\n### Step 2: Test It\n1. Create a new note\n2. Drag an image into the note\n3. Check `04_Resources/`\u2014the image should be there!\n\n---\n\n## Organizing Resources (Optional)\n\nBy default, everything dumps into `04_Resources/` directly. That's fine!\n\nBut if you want more structure:\n\n```\n04_Resources/\n\u251c\u2500\u2500 images/           \u2190 Screenshots, photos\n\u251c\u2500\u2500 pdfs/             \u2190 Articles, handouts\n\u251c\u2500\u2500 datasets/         \u2190 CSV files\n\u251c\u2500\u2500 charts/           \u2190 Exported visualizations\n\u2514\u2500\u2500 misc/             \u2190 Everything else\n```\n\nYou can manually sort files into subfolders. Obsidian will still find them.\n\n---\n\n## Best Practices\n\n### 1. Name Files Clearly\n\u274c `IMG_20260112.png`\n\u2705 `week5_framing_theory_slide.png`\n\n\u274c `download.pdf`\n\u2705 `Entman_1993_Framing.pdf`\n\n### 2. Keep Originals\nIf you edit an image, save the edited version with a new name:\n- `chart_original.png`\n- `chart_edited.png`\n\n### 3. Reference Files in Notes\nWhen you add a resource, create a note about it:\n\n```markdown\n# Week 5 Lecture Slides\n\n**File:** [[week5_slides.pdf]]\n**Date:** January 25, 2026\n**Topic:** Framing Theory\n\n## Key Points\n- [Your notes here]\n```\n\n---\n\n## Linking to Resources in Notes\n\n### Embed an Image\n```markdown\n![Description](image_filename.png)\n```\n\n### Link to a PDF\n```markdown\n[[filename.pdf]]\n```\n\n### Link with Custom Text\n```markdown\n[See the article here](Entman_1993.pdf)\n```\n\n---\n\n## Managing Large Files\n\n### Problem:\nObsidian vaults can get BIG if you store lots of PDFs and images.\n\n### Solutions:\n\n1. **Use Zotero for PDFs** (Recommended)\n   - Store PDFs in Zotero\n   - Link to them from Obsidian\n   - Best of both worlds\n\n2. **Cloud Storage for Big Datasets**\n   - Store large CSV files on OneDrive/Dropbox\n   - Keep a note in `04_Resources/` with the link\n   - Example:\n     ```markdown\n     # Project Dataset\n     **Location:** [OneDrive Link]\n     **Last Updated:** Jan 12, 2026\n     ```\n\n3. **Compress Images**\n   - Use tools like TinyPNG before adding to vault\n   - Reduces file size without losing quality\n\n---\n\n## Syncing Resources Across Devices\n\n### Option 1: Obsidian Sync (Paid)\n- Official Obsidian service\n- $10/month or $96/year\n- Includes version history\n\n### Option 2: Cloud Storage (Free)\n- Store your entire vault in OneDrive/Dropbox/Google Drive\n- Syncs automatically\n- Works great for most users\n\n### Option 3: GitHub (For Advanced Users)\n- Store vault in a GitHub repository\n- Commit and push changes\n- Full version control\n\n---\n\n## Backup Strategy\n\n**Rule:** You don't have a backup until you have **3 copies in 2 locations**.\n\n### Example Backup Plan:\n1. **Copy 1:** Your local computer (primary)\n2. **Copy 2:** OneDrive/Dropbox (cloud)\n3. **Copy 3:** External hard drive (monthly backup)\n\n### Automate It:\n- OneDrive/Dropbox syncs automatically\n- Set a monthly reminder to copy to external drive\n- Test your backups occasionally\n\n---\n\n## Troubleshooting\n\n### \"My image isn't showing!\"\n\u2192 Check that the file is actually in `04_Resources/`\n\u2192 Check the filename spelling (case-sensitive!)\n\u2192 Try using `![[filename.png]]` instead of `![](filename.png)`\n\n### \"I moved a file and the link broke!\"\n\u2192 Obsidian has a \"Detect all file links\" feature in Settings\n\u2192 Or: Find all instances of `[[old_name]]` and replace with `[[new_name]]`\n\n### \"My vault is too big to sync!\"\n\u2192 Move large files to cloud storage\n\u2192 Link to them instead of embedding\n\u2192 Compress images before adding\n\n---\n\n**Pro tip:** This folder is \"set it and forget it.\" Configure it once at the start, then just drag files in. Obsidian handles the rest. \ud83d\uddc2\ufe0f
