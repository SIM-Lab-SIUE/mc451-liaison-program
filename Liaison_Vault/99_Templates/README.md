# \ud83d\udccb Templates - Your Starter Files\n\nThis folder contains **reusable templates** for common tasks and assignments.\n\n## What's Included\n\n### 1. Weekly_Journal_Template.md\n**Use for:** Your weekly reflection journals (due every Monday)\n\n**How to use:**\n1. Copy this file\n2. Paste into `01_Journal/`\n3. Rename to `Week_XX_YourName.md`\n4. Choose one thinking path (Connector/Troubleshooter/Critic)\n5. Delete the other two paths\n6. Fill it out (300-500 words)\n\n---\n\n### 2. Data_Dictionary_Template.md\n**Use for:** Your codebook assignment (Week 8-10)\n\n**How to use:**\n1. Copy this file\n2. Paste into `03_Project/02_Codebook/`\n3. Rename to `Data_Dictionary.md` or `Codebook.md`\n4. Fill in your variables row by row\n5. Document coding rules and edge cases\n\n---\n\n## How Templates Work\n\n### The Copy-Paste Workflow\n\n1. **Right-click** the template file\n2. Select **\"Make a copy\"** or **\"Duplicate\"**\n3. **Move** the copy to the appropriate folder\n4. **Rename** with a descriptive title\n5. **Fill out** the template\n6. **Delete** placeholder text as you go\n\n### Keyboard Shortcut (Faster)\n\n1. Open the template\n2. Press `Ctrl+A` (or `Cmd+A` on Mac) to select all\n3. Press `Ctrl+C` to copy\n4. Create a new note in the target folder\n5. Press `Ctrl+V` to paste\n6. Rename and fill out\n\n---\n\n## Creating Your Own Templates\n\nAs you progress through the course, you might want to create templates for:\n\n- Interview protocols\n- Literature note structure\n- Meeting notes format\n- Data analysis logs\n- Peer review feedback forms\n\nJust create a file here named `[Something]_Template.md` and reuse it!\n\n---\n\n## Template Best Practices\n\n### 1. Use Placeholders\nMake it obvious what needs to be filled in:\n\n\u2705 `[Insert your research question here]`\n\u2705 `**Date:** [YYYY-MM-DD]`\n\u2705 `## [Section Title]`\n\n\u274c Blank spaces without guidance\n\n### 2. Include Instructions\nAdd comments in the template:\n\n```markdown\n## Analysis\n\n<!-- Write 2-3 paragraphs analyzing your findings.\nInclude at least one data visualization. -->\n\n[Your analysis here]\n```\n\nThe `<!-- comment -->` won't show when rendered.\n\n### 3. Pre-Fill Metadata\n\nAdd common tags, headers, and structure:\n\n```markdown\n---\ntitle: \"Week X Journal\"\nauthor: \"Your Name\"\ndate: \"YYYY-MM-DD\"\ntags: [journal, weekX, phaseX]\n---\n```\n\n---\n\n## Advanced: Template Plugin (Optional)\n\nIf you want more power, install the **Templater** plugin:\n\n1. Go to Settings â†’ Community Plugins\n2. Browse and search for \"Templater\"\n3. Install and enable\n4. Set template folder to `99_Templates/`\n\nNow you can:\n- Auto-insert today's date\n- Auto-generate file names\n- Create dynamic templates with variables\n\n### Example: Auto-Date Template\n\n```markdown\n---\ntitle: \"Week X Journal\"\ndate: <% tp.date.now(\"YYYY-MM-DD\") %>\n---\n\n# Week X: [Title]\n\n**Date:** <% tp.date.now(\"MMMM D, YYYY\") %>\n```\n\nTemplater fills in today's date automatically!\n\n---\n\n## Why Templates Matter\n\n### Consistency\nEveryone's journals look the same.
